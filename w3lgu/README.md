# W3Lgu — W3 Language Unit

W3Lgu คือภาษาการทำงานภายในของระบบ W3  
ออกแบบมาเพื่อให้ระบบ “คิด–ตัดสินใจ–ลงมือ” ได้ตรงไปตรงมา  
ไม่เน้นสวย ไม่เน้นสากล แต่เน้น **เร็ว / เบา / แรง / ไม่หลงบริบท**

นี่ไม่ใช่ภาษาเพื่ออธิบาย  
แต่มันคือภาษาเพื่อ “กระทำ”

---

## หลักการอ่าน (Reading Principle)

- อ่านจาก **ซ้าย → ขวา**
- อ่านจาก **บน → ล่าง**
- 1 บรรทัด = 1 เหตุการณ์หรือ 1 ลำดับเหตุการณ์
- ไม่มีการเดา ไม่มี implicit magic

---

## กฎแกนกลาง (Core Rules)

- **ตัวพิมพ์ใหญ่** = คำสั่ง (Command)
- **ตัวพิมพ์เล็ก** = ข้อมูล / สถานะ
- คำสั่งสามารถอยู่ตำแหน่งใดก็ได้ในบรรทัด
- ความหมายเกิดจาก “บริบท + ลำดับ”

---

## เครื่องหมายหลัก (Minimal Symbols)

- `:` เริ่มบริบท
- `;` แยกลำดับเหตุการณ์
- `'` คุณสมบัติ / สถานะ
- `/` เรียกใช้ทรัพยากร
- `>` การส่งต่อ / ผลลัพธ์
- `.` สิ้นสุดการกระทำ

---

## ตัวอย่างพื้นฐาน

```text
P: /gun;SHOOT.

ความหมาย:

P เป็นประธาน

ใช้ gun

ยิง

จบการกระทำ



---

เงื่อนไข & ลูป

LOOP6543: input;ammo>5;input>P'gun;SHOOT
CLOSE6543.

การอ่านเชิงระบบ:

1. เริ่มลูป


2. รับ input


3. ตรวจ ammo


4. ถ้าผ่าน → ส่ง input ให้ P ใช้ gun และ SHOOT


5. ไม่ผ่าน → ออกจากลูป

Chaotic Space Reading (CSR)

โหมดอ่านพิเศษสำหรับสถานการณ์ที่ข้อมูลไม่เรียงเชิงเส้น
ใช้เฉพาะเมื่อจำเป็น

CCSE1:[A];[B];[C]
CCSE2:[1];[2];[3]

ระบบยังอ่าน L→R, U→D

แต่อนุญาตให้ “ข้ามพื้นที่” ภายในขอบเขตที่กำหนด

ใช้กับงานเชิงสัมพันธ์ / ตาราง / map



---

โครงสร้างเชิงระบบ (Conceptual)

W3Lgu ทำงานร่วมกับ Process Layer ของ W3:

REDR — อ่าน / ห่อ / แท็ก

PSP2 — กระจาย Package

DTML — ตรวจ / อนุมัติ / หยุด

MNPS — ประมวลผล

LRC2 — บันทึก / trigger จากความผิดพลาด


ภาษาไม่แทนโมดูล
แต่ทำให้โมดูล “เข้าใจตรงกัน”


---

สิ่งที่ W3Lgu ไม่ใช่

ไม่ใช่ภาษาโปรแกรมทั่วไป

ไม่ใช่ DSL เพื่อการตลาด

ไม่พยายามแทนภาษาอื่น


W3Lgu คือ ภาษาที่ระบบใช้คิดกับตัวเอง


---

สถานะปัจจุบัน

Phase: Experimental / Living Language

Spec ยังไม่ตายตัว

ปิดท้าย

ภาษานี้เกิดจากการลองผิด ล้ม และลุกซ้ำ
ไม่ใช่จากตำรา
แต่จากประสบการณ์ตรงของระบบที่ต้องอยู่รอด

ถ้ามันยังอ่านง่ายในวันที่ระบบซับซ้อน
แปลว่ามันยังเดินถูกทาง

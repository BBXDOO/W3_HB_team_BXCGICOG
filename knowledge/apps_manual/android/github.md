# GitHub (Mobile)

## 1) ทำอะไรได้ (1 บรรทัด)
- ใช้เก็บไฟล์/เอกสาร/ระบบงานของ W3 แบบเป็นศูนย์กลาง (HQ) และย้อนดูประวัติได้

## 2) ใช้กับงานของฉันยังไง
- เก็บ “คู่มือแอป” ที่ฉันทำไว้ → ไม่ต้องอธิบายซ้ำ
- เก็บ Template / SOP / Workflow ของทีม W3 → ทุกคนใช้มาตรฐานเดียวกัน
- ใช้เป็นคลังงานออกแบบ/ระบบ/ไอเดีย → ไม่หาย แม้มือถือพัง

## 3) เริ่มใช้งานด่วน 5 ขั้น
1) เข้าแอป GitHub → ไปที่ repo ที่ใช้งาน (เช่น `W3_HB_team_BXCGICOG`)
2) เช็ค branch ให้ถูกต้อง: ✅ `refactor/v0.2` (ห้ามไปยุ่ง `main`)
3) เข้าโฟลเดอร์ที่ต้องการ → กด `Add file` เพื่อเพิ่มไฟล์
4) สร้างไฟล์ `.md` แล้ววางเนื้อหา/แก้ไข → กด Commit
5) กลับไปที่ `INDEX.md` แล้วเพิ่มลิงก์ให้ไฟล์ใหม่ เพื่อให้หาเจอง่าย

## 4) เมนู/คำสั่งสำคัญ
- **Branch selector**: เลือกสาขาที่ทำงาน (เช่น `refactor/v0.2`)
- **Add file**: สร้างไฟล์ใหม่ / อัปโหลดไฟล์
- **Edit (✏️)**: แก้ไขไฟล์
- **Commit changes**: บันทึกการเปลี่ยนแปลง (เหมือน “เซ็นงานเข้าระบบ”)
- **History / Blame**: ดูว่าใครแก้เมื่อไหร่ และแก้ตรงไหน
- **Raw**: ดูไฟล์แบบข้อความล้วน (คัดลอกง่าย)
- **Copy path**: คัดลอกพาธไฟล์ (ทำลิงก์ใน INDEX ได้ไว)

## 5) ทิป (กันพลาด)
- ใช้ลิงก์แบบ Relative เสมอใน `INDEX.md` เช่น:
  - `[GitHub](android/github.md)`
  เพื่อไม่ให้พังเวลาเปลี่ยน branch
- ตั้งชื่อไฟล์ให้เป็นมาตรฐานตัวเล็ก + ขีดล่าง เช่น:
  - `google_docs.md`, `pdf_tools.md`
- Commit message ให้สื่อความชัด:
  - `feat(knowledge): add github manual`
  - `docs(apps): update github tips`
- ทำงานแบบ “เพิ่มน้อยแต่ชัวร์” (วันละ 1 ไฟล์ก็พอ)

## 6) ปัญหาที่พบบ่อย + วิธีแก้
- อาการ: เผลออยู่ branch ผิด (เช่น main)
  วิธีแก้: กลับไปเลือก branch `refactor/v0.2` ก่อนทำทุกครั้ง

- อาการ: กดลิงก์ใน INDEX แล้ว 404
  วิธีแก้: ตรวจว่ามีไฟล์ปลายทางจริงไหม + path ถูกไหม (ตัวพิมพ์เล็ก/ใหญ่สำคัญ)

- อาการ: แก้ไฟล์แล้วไม่รู้ว่าบันทึกหรือยัง
  วิธีแก้: ต้องกด `Commit changes` ทุกครั้ง ไม่งั้นไม่ถูกบันทึก

- อาการ: หาไฟล์ไม่เจอ
  วิธีแก้: ใช้ช่อง Search/Go to file หรือทำสารบัญใน `INDEX.md` ให้ครบ
